Profile Page

Technical Description

The profile page is built using HTML and CSS with a structured layout that includes a navigation bar, main content area, and footer. The page layout is achieved using flexbox and grid-based structures to ensure responsiveness. The profile information is displayed using multiple card-style sections (.card1, .card2, .card3, etc.), which are styled with custom borders, background colors, and box shadows. A progress bar is used to indicate skill levels, implemented with div elements styled dynamically. The form elements for contact purposes are styled for a modern and clean appearance. The site maintains consistency by utilizing CSS media queries for mobile responsiveness.

Accessibility

The profile page is designed with accessibility in mind, ensuring that users with different needs can navigate and interact with the content effectively. High-contrast colors are used for text and background elements to improve readability, especially for visually impaired users. Images and icons include alt attributes, allowing screen readers to describe the content to users who rely on assistive technologies.
The page follows a proper heading hierarchy, ensuring logical content flow for screen readers. Interactive elements such as buttons and navigation links have hover effects to provide visual feedback, making the interface more user-friendly. The fixed navigation bar ensures easy access to different sections of the website, reducing the need for excessive scrolling.
Form inputs are labeled clearly, helping users understand the purpose of each field, which is particularly useful for those using screen readers. Additionally, CSS media queries make the page responsive, ensuring accessibility across various devices and screen sizes.

Sitemap Page

Technical Description

The sitemap page is developed using HTML, CSS, and SVG to visually represent the website structure. The navigation bar at the top is styled with a fixed position for easy access. The main content area features an SVG-based diagram that maps out the site's hierarchy using circles, ellipses, and lines, with each representing different pages and sections. Interactive elements are implemented using anchor () tags within SVG, allowing users to navigate by clicking on different nodes. The CSS styles ensure a visually appealing design with hover effects on elements to enhance user engagement. Media queries ensure that the sitemap remains responsive across various screen sizes.

Accessibility

The sitemap page is designed to be inclusive and user-friendly. The text inside the SVG elements is bold and properly aligned, making it easier to read. Each interactive node has hover effects to provide visual feedback, aiding users in navigation. The alt attributes are used for icons, and a high-contrast color scheme ensures better visibility. The navigation bar remains fixed, improving accessibility for users with motor impairments by reducing excessive scrolling. Additionally, the page structure follows a clear hierarchy, making it easier for screen readers to interpret and guide visually impaired users through the websiteâ€™s structure.

Content Page

Technical Description

The webpage is built using HTML and CSS. The structure follows a responsive layout utilizing div elements with classes such as row, column, and card for content organization. A background image is applied to a div instead of using the () tag to enhance responsiveness. The navigation bar is fixed at the top and includes interactive hover effects. The footer contains structured links for navigation. The CSS styles include a flexible grid-based layout with media queries ensuring responsiveness on different screen sizes.

Accessibility

The webpage incorporates accessibility features such as high-contrast text and background colors for readability. Anchor tags use :hover effects for better interactivity. Images include alt attributes for screen readers. The website follows a proper heading hierarchy, ensuring better navigation for assistive technologies. Buttons and interactive elements have visible hover states to improve user experience.

Challenges and Lessons Learned

IOne of the primary challenges in developing the profile and sitemap pages was ensuring proper layout and alignment across different screen sizes. Initially, elements such as profile cards and the SVG sitemap structure did not scale well on smaller screens. To solve this, I implemented CSS media queries for breakpoints at 1024px, 900px, and 768px, adjusting flex properties, widths, and text sizes accordingly.Another challenge was making the SVG elements interactive and readable. Text within circles was initially misaligned, so I learned how to use text-anchor and dominant-baseline properties to properly center text in SVG elements. Additionally, I improved my understanding of hover effects and how to create visually distinct UI elements while maintaining a clean and accessible design.Through this process, I enhanced my skills in responsive design, SVG handling, and accessibility-focused styling.

Compliance

To align with Janet regulations and WCAG accessibility standards, I implemented the following best practices:

  • Ensured semantic HTML structure, making navigation and content easier for screen readers.
  • Validated HTML and CSS using W3C validation tools to ensure error-free code.
  • Added alt attributes for all images and icons to improve accessibility.
  • Optimized SVG elements for accessibility, ensuring interactive components remain usable across different devices.
