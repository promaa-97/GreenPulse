Splash Page

Technical Description

The splash page serves as a brief introductory screen before automatically redirecting users to the volunteer page after 4 seconds. It features a full-screen background image, a project logo, a mission statement, and a table listing the team members. The page uses Google Fonts for typography and links to an external CSS file for styling. The redirection is handled using a meta refresh tag in the head section.

Accessibility

To ensure accessibility, the page uses semantic HTML for better screen reader support, includes alt text for images, and applies a viewport meta tag for mobile responsiveness. The text is designed with high contrast for readability. However, since the page auto-redirects, adding a progress indicator or a manual "Skip" button could further improve accessibility for users who need more time to read the content.

Volunteer Page

Technical Description

the volunteer page is built using HTML and CSS, featuring a fixed navigation bar, interactive volunteer program cards, an application form, and a structured footer with useful links. The design is responsive, utilizing media queries to ensure compatibility across different screen sizes. It incorporates hover effects, a clean layout, and an eco-friendly color scheme to enhance user experience.

Accessibility

While the page follows basic accessibility principles, improvements can be made by adding ARIA labels to form inputs, ensuring keyboard navigability, and enhancing color contrast for readability. Implementing focus indicators, descriptive alt text for images, and proper form validation will make the page more inclusive and user-friendly for individuals with disabilities.

Content Page

Technical Description

The HTML structure is well-organized, with a navigation bar, content sections, and a footer. The CSS uses flexbox for layout, hover effects for interaction, and media queries for responsiveness. However, some elements have fixed margins and heights, which may affect mobile usability. Images are added through CSS backgrounds, which could impact accessibility and search engine optimization.

Accessibility

The webpage has a clear navigation structure and good text readability, but some color contrasts and hover effects may need improvement. Keyboard navigation works, but focus indicators are missing, making it harder for users who rely on the keyboard. Some images lack alternative text, which could affect screen readers. The page is responsive, but some large margins may create extra white space on smaller screens. Using more meaningful HTML elements instead of generic containers could improve accessibility.

Challenges and Lessons Learned

One of the main challenges in this webpage was ensuring accessibility, as missing focus indicators made keyboard navigation difficult, and background images lacked alternative text for screen readers. Maintaining a responsive design was also challenging due to fixed margins and heights that could cause layout issues on smaller screens. Additionally, hover effects that changed font sizes led to layout shifts, affecting readability. Through this project, we learned the importance of using semantic HTML for better structure, ensuring proper color contrast for readability, and optimizing CSS for performance and maintainability. Prioritizing accessibility and responsive design improves user experience across all devices.
