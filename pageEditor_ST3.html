Feedback Page

Technical Description

Created using HTML, CSS, and JavaScript. Fieldset tags organize related form sections. Anchor tags connect the navigation bar. The video background includes autoplay and loop attributes. Various input types, like file upload and date pickers, capture user data. Emojis represent ratings. JavaScript is used to count remaining characters in the feedback field. Submit buttons ensure smooth form completion.

Accessibility

Added :hover effects for interactive navigation links, buttons, and rating emojis. Default font and background color enhance readability. Implemented fixed-position video background. Form design includes padding and rounded borders for better accessibility. Applied flexible spacing for responsive layouts using media queries. Inputs and text areas styled for uniform appearance. Alt attributes and clear labels improve usability for screen readers.

Team Page

Technical Description

Created using HTML and CSS. Used div tags to organize the layout for better structure. Added images with img tags and alt attributes for accessibility. The navigation bar is fixed for consistent display. Used anchor tags to link to other pages and added styling for hover effects. Organized student details into separate sections for clarity and readability.

Accessibility

Used a default background color to improve readability. Added :hover effects on navigation links, student details, and profile images to enhance interactivity. Styled borders, buttons, and text using CSS for a better user experience. Applied high-contrast colors between text and backgrounds for clarity. Included alt attributes for all images to support screen readers. Organized headings and sections with proper structure for assistive technologies.

Content Page

Technical Description

Created using HTML and CSS. Used div tags to organize content effectively. Designed a side menu with anchor tags for easy navigation between sections. Styled headings and paragraphs using CSS with custom colors and font sizes. Applied background images to div tags for responsiveness. Added a back-to-top button to navigate to the top. Styled cards with proper padding, margins, and border-radius using CSS.

Accessibility

Applied :hover effects for interactive links in the side menu. Used dark blue and white as the main color theme. Gave 20% margin on both sides of the container to improve readability. Added alt attributes to all images for screen readers. Used high-contrast colors between text and backgrounds for visually impaired users. Maintained proper heading hierarchy for assistive technologies.

Challenges and Lessons Learned

Initially, the navigation bar and content layout were inconsistent on smaller screens. To fix this, I used media queries for 1024px and 768px screen widths and adjusted flex properties and spacing. I learned how critical responsive design is for usability. I also discovered the value of hover effects and proper alt attributes in making pages both interactive and accessible.

Compliance

To ensure compliance with Janet regulations and WCAG standards, I followed best practices for web development:

  • Used proper syntax and validated code using the W3C HTML and CSS validator tools.
  • Ensured no user data is stored insecurely and followed privacy best practices.
  • Maintained a logical page structure with clear, accessible navigation links.
  • Used responsive design principles to make the website accessible across devices.
